CSS Units: Complete Guide with Interactive Examples

1. Percentage (%) - Parent-Relative Sizing

How it works: Percentage values are calculated relative to the parent element's dimensions.

Formula: Child size = (Percentage / 100) × Parent size
.parent { width: 400px; height: 200px; } .child-50 { width: 50%; /* 200px */ height: 40%; /* 80px */ } .child-100 { width: 100%; /* 400px */ height: 30%; /* 60px */ }
Parent: 400px × 200px
50% × 40% = 200px × 80px
100% × 30% = 400px × 60px
✅ Best for: Layout containers, responsive grids, flexible components that should scale with their parent
⚠️ Watch out: If parent has no explicit width/height, percentage might not work as expected

2. Viewport Units (vw/vh) - Screen-Relative Sizing

How it works: Based on the viewport (browser window) dimensions.
1vw = 1% of viewport width | 1vh = 1% of viewport height

Example: On a 1200px wide screen, 50vw = 600px
.hero-text { font-size: 4vw; /* Scales with screen width */ } .full-height { height: 100vh; /* Always full screen height */ } .half-width { width: 50vw; /* Always half screen width */ }
4vw Text

This box is 50vw wide, 20vh tall. Resize your browser to see it change!

✅ Best for: Hero sections, full-screen layouts, responsive typography, landing page elements
⚠️ Watch out: Can cause text to become too small on mobile or too large on wide screens. Always set min/max limits.

3. REM - Root Font-Size Relative

How it works: Always relative to the root element's (html) font-size.
Default: 1rem = 16px (browser default)
Consistent: 1rem is always the same size throughout your entire website
html { font-size: 16px; /* Root size */ } .container { margin: 1.5rem; /* 24px */ padding: 2rem; /* 32px */ font-size: 1.5rem; /* 24px */ } /* Always consistent across site */
1.5rem Text (24px)

This container has 2rem padding (32px) and 1.5rem margin (24px)

REM values are always consistent - they don't compound!

✅ Best for: Consistent spacing, typography scales, margins/padding, component sizing
Root Font Size 1rem = 1.5rem = 2rem =
16px (default) 16px 24px 32px
18px 18px 27px 36px
14px 14px 21px 28px

4. EM - Parent Font-Size Relative

How it works: Relative to the parent element's font-size.
Compounding: If nested, em values multiply (can be tricky!)
Context-aware: Same em value can result in different pixel sizes
.parent { font-size: 18px; } .child { font-size: 1.2em; /* 21.6px */ padding: 0.5em; /* 10.8px */ } .nested { font-size: 1.2em; /* 25.92px! */ /* Compounding: 21.6px * 1.2 */ }
Parent: 18px font-size
Child: 1.2em = 21.6px
Nested: 1.2em = 25.92px (compounded!)
✅ Best for: Component-specific spacing, scalable components, contextual sizing
⚠️ Watch out: Compounding effect in nested elements can cause unexpected sizing

When to Use Each Unit

Unit Best Use Cases Avoid When Example Scenario
% Layout containers, responsive grids, flexible widths Fixed-size elements, borders, small spacing Sidebar: width: 25%; Main: width: 75%
vw/vh Hero sections, full-screen layouts, responsive typography Small text, precise measurements Hero title: font-size: 5vw
rem Consistent spacing, typography, component sizes Parent-dependent sizing Button padding: 1rem 2rem
em Contextual spacing, scalable components Global consistency needed Icon size relative to text: width: 1.2em

🎯 Pro Tips for Responsive Design:

  • Layout: Use % for flexible containers, CSS Grid for complex layouts
  • Typography: Use rem for consistent sizes, em for contextual scaling
  • Spacing: rem for consistent spacing system, em for component-specific
  • Responsive Elements: vw/vh for full-screen components, % for flexible sizing
  • Media Queries: Use em-based breakpoints for better accessibility

Real-World Example: Responsive Card Component

.card { width: 100%; /* % - Flexible width */ max-width: 20rem; /* rem - Consistent max size */ margin: 1rem; /* rem - Consistent spacing */ padding: 1.5rem; /* rem - Consistent internal spacing */ } .card-title { font-size: 1.5rem; /* rem - Consistent typography */ margin-bottom: 1rem; /* rem - Consistent spacing */ } .card-icon { width: 1.2em; /* em - Scales with text */ height: 1.2em; /* em - Scales with text */ } @media (max-width: 48em) { /* em-based breakpoint */ .card { width: 90vw; /* vw - Responsive to viewport */ margin: 0.5rem auto; /* rem - Smaller consistent margin */ } }
